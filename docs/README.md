# ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ ВЕБ-РЕСУРСА «FOREST RANGER»

Настоящий документ содержит описание архитектуры, структуры файлов, интерфейсных решений и стилистического оформления статического веб-сайта проекта **«Forest Ranger»**. Веб-сайт служит информационной платформой для презентации разрабатываемой мобильной игры, координации проектной группы и публикации отчетов о ходе разработки.

---

## 1. Общие сведения о проекте

**Forest Ranger** — это мобильная игра в жанре экшн-приключения. Игрок выступает в роли егеря, охраняющего заповедную зону тайги от браконьеров, экологических катастроф и лесных пожаров. 

Цель создания веб-сайта — агрегация всей информации о проекте, представление команды разработки, фиксация хронологии создания игры (дневник разработчиков) и предоставление доступа к смежным ресурсам.

---

## 2. Технологический стек

При проектировании и разработке веб-сайта использовались исключительно стандартные веб-технологии без привлечения тяжеловесных фреймворков, что гарантирует максимальную скорость загрузки страниц на любых устройствах:
* **HTML5:** Семантическая разметка контента (`<header>`, `<nav>`, `<section>`, `<footer>`, списки, формы).
* **CSS3:** Каскадные таблицы стилей, включая современные механизмы адаптивной раскладки (Flexbox, CSS Grid), позиционирование (`sticky`), трансформации (`transform: scale`) и плавные переходы (`transition`).
* **Git:** Система контроля версий для отслеживания изменений и совместной работы.
* **Markdown:** Формат документирования проекта.

## 4. Спецификация веб-страниц

Каждая страница сайта разработана с учетом требований к доступности, семантике и адаптивности. Шапка (`<header>`) с навигационным меню и подвал (`<footer>`) идентичны на всех страницах, что обеспечивает целостность пользовательского опыта.

### 4.1. Главная страница (`index.html`)
Предназначена для первичного ознакомления пользователя с проектом.

* **Навигационная панель (`<nav>`):** Ссылка на текущую страницу имеет класс `active` (подсвечивается зеленым цветом и подчеркивается).
* **Главный баннер (`.hero-banner`):** Полноэкранный блок с фоновым изображением тайги, содержащий приветственный заголовок и девиз игры. Использует эффект внутреннего затенения (`box-shadow: inset`) для повышения читаемости белого текста.
* **Аннотация проекта:** Краткое описание концепции, жанра, целевой платформы (RuStore) и модели монетизации (Pay-to-play).
* **Интерактивный баннер журнала разработки (`.image-link`):** Ссылка-изображение, ведущая на страницу дневников. При наведении курсора плавно масштабируется (`transform: scale(1.02)`), привлекая внимание пользователя.
* **Сетка участников (`.card-grid`):** Реализована на базе CSS Grid. Содержит 17 персональных карточек участников команды с указанием их ролей. Карточка руководителя проекта (Дмитриева К.Д.) выделена информационно как Тимлид и Главный геймдизайнер.

### 4.2. Страница «О проекте» (`about.html`)
Содержит подробное описание игрового процесса, ключевых механик и сюжетной линии.

* **Игровые механики:** Представлены в виде маркированного списка, детально описывающего патрулирование, установку фотоловушек, нелетальный характер боевой системы (ближний бой, шокеры, перцовые баллончики), а также поведение уникального спутника героя — красного волка.
* **Сюжетный блок:** Текстовый раздел, раскрывающий детективную линию расследования гибели отца главного героя, криминальный синдикат браконьеров и финальный сюжетный поворот с предательством близкого друга.
* **Техническая сводка:** Выделенный информационный блок с контрастной левой границей (`border-left: 4px solid #2c3e50`) и серым фоном, содержащий ключевые параметры дистрибуции (целевая аудитория, мобильные платформы Android/iOS).

### 4.3. Журнал разработки (`journal.html`)
Выполняет роль блога проекта (Changelog) и отражает динамику работы над игрой. Содержит 8 последовательных записей, оформленных в виде карточек постов (`.post`):

## 5. Архитектура стилей (`style.css`)

Оформление сайта выполнено в строгой экологической цветовой гамме (темно-синие, глубокие серые и природно-зеленые тона).

### 5.1. Цветовая палитра системы
* **Основной фон сайта:** Светло-серый с зеленым оттенком (`#f4f7f6`).
* **Фон контентных блоков:** Чистый белый (`white`) для обеспечения контрастности.
* **Цвет основного текста:** Темно-серый (`#333`) для снижения утомляемости глаз.
* **Цвет шапки и подвала:** Глубокий темно-синий (`#1b262c`).
* **Акцентный цвет (интерактивные элементы, границы, активные ссылки):** Натуральный зеленый (`#4caf50`).
* **Дополнительный темный акцент (заголовки, рамки):** Грифельно-серый (`#2c3e50`).

### 5.2. Ключевые CSS-решения
* **Сброс стилей:**
  ```css
  * {
      margin: 0;
      padding: 0;
      box-sizing: border-box;
  }
  ```
  Применяется ко всем элементам для предотвращения расхождения кроссбраузерной верстки из-за дефолтных отступов.

* **Фиксированная навигация:**
  Шапка сайта удерживается в верхней части экрана при прокрутке благодаря свойству `position: sticky; top: 0; z-index: 100;`, что облегчает переход между страницами.

* **Адаптивная сетка Grid:**
  Список участников команды автоматически перестраивает количество колонок в зависимости от ширины экрана устройства:
  ```css
  .card-grid {
      display: grid;
      grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
      gap: 15px;
  }
  ```
  Это исключает необходимость написания избыточных медиа-запросов (`@media`) для мобильных устройств.

* **Интерактивные анимации (Transitions):**
  При наведении на ссылки и кликабельные изображения срабатывает плавное изменение цвета и масштаба:
  ```css
  transition: color 0.3s;
  transition: transform 0.3s ease, box-shadow 0.3s ease;
  ```

## 6. Инструкция для разработчиков по ведению проекта

Для внесения изменений в веб-ресурс «Forest Ranger» необходимо соблювать следующий регламент работы с Git-репозиторием:

1. **Клонирование репозитория:**
   ```bash
   git clone <URL_репозитория>
   ```
2. **Создание рабочей ветки** для внесения изменений (например, добавления новой записи в журнал):
   ```bash
   git checkout -b feature/add-journal-post
   ```
3. **Локальное тестирование:**
   Открыть измененные файлы `.html` в браузере, проверить отображение стилей через панель разработчика (DevTools — вызвать клавишей `F12`).
4. **Фиксация изменений:**
   Добавить файлы в индекс и создать коммит с осмысленным описанием по стандарту Conventional Commits:
   ```bash
   git add journal.html
   git commit -m "docs: add post 9 to development journal"
   ```
5. **Отправка изменений на сервер:**
   ```bash
   git push origin feature/add-journal-post
   ```
